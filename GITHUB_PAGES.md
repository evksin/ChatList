# Инструкция по настройке GitHub Pages

## Быстрый старт

### 1. Подготовка файлов

1. Создайте файл `index.html` в корне репозитория (или в папке `docs/`)
2. Используйте готовый шаблон из `index.html` (лендинг для проекта)

### 2. Настройка GitHub Pages

#### Вариант A: Использовать корневую папку

1. Перейдите в **Settings** → **Pages** вашего репозитория
2. В разделе **Source** выберите:
   - **Branch**: `main` (или `master`)
   - **Folder**: `/ (root)`
3. Нажмите **Save**

#### Вариант B: Использовать папку docs/ (рекомендуется)

1. Создайте папку `docs/` в корне репозитория
2. Поместите `index.html` в папку `docs/`
3. Перейдите в **Settings** → **Pages**
4. В разделе **Source** выберите:
   - **Branch**: `main`
   - **Folder**: `/docs`
5. Нажмите **Save**

### 3. Проверка

После сохранения настроек GitHub автоматически опубликует сайт. Обычно это занимает 1-2 минуты.

Ваш сайт будет доступен по адресу:
```
https://yourusername.github.io/chatlist/
```

## Обновление сайта

После каждого изменения в `index.html` или файлах в папке `docs/`:

1. Закоммитьте изменения:
   ```powershell
   git add index.html
   git commit -m "Update landing page"
   git push
   ```

2. GitHub автоматически обновит сайт (обычно через 1-2 минуты)

## Кастомизация

### Изменение темы

Отредактируйте файл `index.html`:
- Измените цвета в секции `<style>`
- Обновите текст и изображения
- Добавьте свои секции

### Добавление скриншотов

1. Создайте папку `docs/images/` (или `images/` в корне)
2. Поместите скриншоты в эту папку
3. Обновите ссылки в `index.html`:
   ```html
   <img src="images/screenshot1.png" alt="Screenshot">
   ```

### Добавление документации

Создайте дополнительные HTML-страницы:
- `docs/features.html` - Описание функций
- `docs/installation.html` - Инструкция по установке
- `docs/faq.html` - Часто задаваемые вопросы

И добавьте ссылки на них в `index.html`.

## Рекомендации

1. **Мобильная версия**: Убедитесь, что сайт корректно отображается на мобильных устройствах
2. **SEO**: Добавьте мета-теги для поисковых систем
3. **Аналитика**: Добавьте Google Analytics или другую аналитику (опционально)
4. **Обновления**: Регулярно обновляйте информацию о версиях и функциях

## Troubleshooting

### Сайт не обновляется

1. Проверьте, что файлы закоммичены и запушены
2. Убедитесь, что выбрана правильная ветка в настройках Pages
3. Подождите несколько минут (GitHub может кэшировать)

### Ошибка 404

1. Убедитесь, что файл `index.html` находится в правильной папке
2. Проверьте настройки Pages (правильная ветка и папка)
3. Проверьте, что файл называется именно `index.html` (с маленькой буквы)

### Проблемы с изображениями

1. Используйте относительные пути: `images/photo.png` вместо абсолютных
2. Убедитесь, что файлы изображений закоммичены в репозиторий
3. Проверьте размер файлов (большие изображения могут загружаться медленно)
